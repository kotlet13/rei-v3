# REI dynamics from Eros comments

Status: paraphrased canonical notes from `Eros - pogovori.pdf`.

## High-priority rules for REI-v3

### 1. Only Racio listens to words

The model must treat verbal output as Racio-compatible. Emocio and Instinkt may influence the result, but their LLM text is a translation.

Implementation rule:
- Racio: `translated_by_racio=false`
- Emocio: `translated_by_racio=true`
- Instinkt: `translated_by_racio=true`

### 2. Same behavior can have different processor origins

A visible action does not prove which processor produced it. The important question is not only "what happened", but "how did the mind get there?"

Regression scenario:
- "I do not want to attend the meeting."
- Racio origin: low utility, bad timing, poor evidence, inefficient sequence.
- Emocio origin: humiliation, dead scene, status wound, lack of recognition, broken image.
- Instinkt origin: distrust, exposure, body alarm, boundary issue, attachment/loss risk.

### 3. Fear is not solved by silencing Instinkt

For fear/fobia-like signals, Racio-only distraction can mute the conscious conflict temporarily but does not address Instinkt. Instinkt needs attention to the fear signal, safety conditions, and translation of the underlying boundary/loss issue.

### 4. Emocio anger comes from broken desired image

When the image that "should be" cannot become the current image, Emocio can produce anger. Racio may later explain the anger incorrectly.

### 5. Acceptance is not happiness

Acceptance is the reduced conflict/cooperation state among processors. A person can have hard circumstances and still have acceptance, or have comfortable circumstances and still be non-accepting.

### 6. Character is stable hierarchy; state is dynamic

Character is the relatively stable relation of processor power. State, acceptance, conflict, trigger, and current pressure are dynamic.
